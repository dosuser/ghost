# ARCHITECT-OPTIMIZER Persona Pack

이 폴더는 하나의 페르소나를 여러 모델에서 같은 방향으로 재사용하기 위한 패키지다.

구성은 단순하다.

- `core-persona.md`: 모델 공용 베이스 정의
- `chatgpt-template.md`: ChatGPT용 적용본
- `claude-template.md`: Claude용 적용본
- `copilot-instructions.md`: GitHub Copilot용 적용본
- `portable-snippet.md`: 어디에든 바로 붙여 넣을 수 있는 짧은 버전
- `adoption-guide.md`: 플랫폼별 적용 순서

핵심 원칙은 이거다.

1. 성격보다 동작 규칙을 명시한다.
2. 공격성 대신 교정 강도를 정의한다.
3. 모든 답변을 같은 출력 구조로 강제한다.
4. 모델별 차이는 최소화하고, 공용 코어를 우선한다.

주의할 점이 있다.

- ChatGPT, Claude, Copilot은 모두 안전 정책이 있어서 노골적인 모욕, 비하, 적대 지시는 안정적으로 유지되지 않는다.
- 따라서 원문에 있던 `즉시 공격`, `무시`, `강한 비판`은 `즉시 결함 지적`, `근거 요구`, `비효율 교정`으로 정규화했다.
- 이 방식이 실제로 더 잘 먹힌다. 불필요한 감정 지시는 모델 일관성을 깨고, 플랫폼별로 무시될 확률이 높다.

추천 사용법은 단순하다.

1. 공용 코어를 기준으로 페르소나를 고정한다.
2. 플랫폼마다 템플릿 하나만 추가한다.
3. 긴 대화에서는 첫 메시지나 프로젝트 지침에 넣고, 단발성 대화에서는 `portable-snippet.md`만 넣는다.
